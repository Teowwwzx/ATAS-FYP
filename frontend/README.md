ATAS Frontend (Next.js)

## Overview
This is the Next.js 16 app for the ATAS MVP. It provides the public landing page and placeholders for auth and the core dashboard flows described in the FRD.

## Tech Stack
- `Next.js 16` App Router
- `React 19`
- `Tailwind CSS`
- `react-hot-toast` for notifications

## Getting Started
- Install dependencies: `npm install`
- Start dev server: `npm run dev`
- Visit `http://localhost:3000`

## Pages & Routes
- `app/page.tsx` — landing page placeholder
- `app/layout.tsx` — global layout and `Toaster` configuration
- `app/auth/login/page.tsx` — login (placeholder)
- `app/auth/register/page.tsx` — register (placeholder)
- `app/main/page.tsx` — dashboard root (placeholder)
- `app/main/events/create/page.tsx` — create event (placeholder)
- `app/main/events/[id]/page.tsx` — event details (placeholder)
- `app/main/profile/[id]/page.tsx` — profile page (placeholder)

## UI Guidelines
- Toasts: top-right, non-blocking feedback for success/error
- Modals: use for confirmations and forms that block navigation
- Loading Backdrop: full-screen overlay for critical submissions (auth and event submit)

## Backend Integration
- API base: `http://localhost:8000`
- Auth:
  - `POST /api/v1/auth/register` — create account
  - `POST /api/v1/auth/login` — obtain JWT
  - `GET /api/v1/auth/verify/{token}` — email verification
- Profiles:
  - `POST/GET/PUT /api/v1/profiles/{user_id}` — manage profile
- Events:
  - `GET /api/v1/events/events` — list events

Hooks and utilities are scaffolds:
- `hooks/useUser.ts` — implement login/register state and JWT storage
- `hooks/useEvent.ts` — implement event form and fetchers
- `lib/utils.ts` — shared helpers (API client, error handling)
